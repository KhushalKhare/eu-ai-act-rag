# EU AI Act RAG Assistant

A corrective, self-evaluating RAG system for querying the EU AI Act — built to demonstrate agentic retrieval, not just search-and-summarize.

**Not legal advice.** Answers are grounded only in the ingested article text and cite the specific Article/clause. Always verify against the official text at [EUR-Lex](https://eur-lex.europa.eu/legal-content/EN/TXT/HTML/?uri=OJ%3AL_202401689).

## Why this isn't "just RAG"

Most RAG demos retrieve once and hope the chunks are relevant. This system grades its own retrieval, and if the grader says the retrieved text doesn't actually answer the question, it rewrites the search query and tries again — up to 3 times — before honestly admitting it doesn't have an answer, rather than generating a confident-sounding hallucination.

## Architecture

```
User question
      |
      v
+-----------------------+
|  Retrieve chunks       |  Chroma + MiniLM embeddings, top 3
+-----------------------+
      |
      v
+-----------------------+
|  Grade relevance       |  Groq Llama 3.3 70B judges the chunks
+-----------------------+
      |
      +-- fails --> +-----------------------+
      |              |  Rewrite query         |  reformulates using the fail reason
      |              +-----------------------+
      |                       |
      |                loop back to Retrieve (max 3 attempts)
      |                       |
      +-- passes / max reached ------------->  |
                                                v
                                  +-----------------------+
                                  |  Generate answer        |  cites Article + clause
                                  +-----------------------+
                                                |
                                                v
                                        Final answer to user
```

Built with **LangGraph** (state machine + conditional routing), **Groq** (Llama 3.3 70B for grading/rewriting/generation), **Chroma** + **sentence-transformers** (local vector search), **FastAPI** (backend), and a vanilla HTML/JS chat frontend.

## How the corrective loop works

1. **Retrieve** — embed the query, pull top-3 chunks from Chroma
2. **Grade** — an LLM judges whether the retrieved chunks actually answer the question (not just topically similar)
3. **Rewrite** (if failed) — reformulate the query using the grader's stated reason for failure, then retry retrieval — capped at 3 attempts to avoid infinite loops on genuinely unanswerable questions
4. **Generate** — produce an answer that cites the specific Article and clause, instructed to only claim what's actually supported by the retrieved text

## Chunking strategy

Legal text has real structural boundaries that generic fixed-size chunking ignores. This pipeline detects and preserves them:
- **Numbered definitions** (Article 3: `(1)`, `(2)`, `(3)`...) — one chunk per definition
- **Lettered clauses** (Article 5: `(a)`, `(b)`, `(c)`...) — one chunk per clause, so a query about biometric ID retrieves exactly Article 5(h), not a blended fragment
- **Fixed-size fallback** for unstructured prose, with overlap to avoid losing context at boundaries

## Evaluation

Since `ragas` had an unresolvable dependency conflict with `langchain-groq` in this environment (both require incompatible versions of `langchain-core`), evaluation is a custom LLM-as-judge harness (`src/eval.py`) scoring two dimensions per question:

- **Faithfulness** — does the answer only claim things actually supported by the retrieved text?
- **Relevancy** — does the answer actually address what was asked (including honestly saying "not covered" when true)?

The eval set (`eval/qa_set.jsonl`) is stratified across categories — definitional, prohibited-practice, scope, multi-hop, adversarial paraphrase, false-premise, and deliberately unanswerable questions — so a single low score points at *which* failure mode, not just "something's wrong."

## Setup

```bash
git clone <your-repo-url>
cd eu-ai-act-rag
python -m venv .venv
.venv\Scripts\activate        # Windows
pip install -r requirements.txt

cp .env.example .env          # then fill in your GROQ_API_KEY

python src/ingest.py          # scrape source articles
python src/chunking.py        # split into retrievable chunks
python src/embed.py           # embed into Chroma
```

Run the API + frontend:
```bash
uvicorn src.api:app --reload
```
Open `http://localhost:8000`.

Run the eval:
```bash
python src/eval.py
```

## Known limitations

- Corpus currently covers Articles 1–7 (general provisions, prohibited practices, high-risk classification rules) — not the full Act
- Chunking's numbered/lettered-clause detection is regex-based and tuned for this corpus; scaling to all 113 articles would benefit from HTML-structure-aware parsing instead
- Faithfulness/relevancy scoring uses the same model family as generation (Llama 3.3 70B via Groq) as judge — an independent judge model would reduce self-grading bias in a production setting

## Disclaimer

This tool provides informational summaries of the EU AI Act and is not legal advice. User queries are sent to Groq's API for processing. No queries are persisted server-side by default.
